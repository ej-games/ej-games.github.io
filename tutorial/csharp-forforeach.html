For vs Foreach

This is a C# (with references to Unity) tutorial about for and foreach loops. This means that it will be able to be followed in vanilla C#, but will also contain context for Unity users.

What is a 'for' loop?

A for loop is an example of a definite iteration.
  • Definite means it repeats a certain number of times
  • Iteration is just another word for looping
In C#, it is a bit more complicated than other languages, for example Python: 
for i in range(10):
	print(str(i))
versus C#: 
for(int i = 0; i < 10; i++) {
	Console.Writeline(i.ToString());
}
Oh, by the way, that's how you make a For loop in C#.

The int i = 0; means 'create a variable, named i, with a value of 0'.
The i < 10; means 'keep looping while i is less than 10'.
The i++ means 'add 1 after each loop'.

What is a 'foreach' loop?

A foreach loop iterates through an array, instead of just a set of numbers.

If you have:
string[] names = new string[] {"John Gold", "David Silva", "Chris Steele", "Joseph Iron"};


then you might use: 
foreach(string name in names) {
	if(name == "John Gold") {
		Console.WriteLine("John Gold has been found!");
	}
}
This might be considered easier and simpler than using a for loop, because you don't have to keep doing names[i] instead of just name.

Just in case you're wondering, the for loop would look like this: 
for(int i = 0; i < names.length; i++) {
	if(names[i] == "John Gold") {
		Console.WriteLine("John Gold has been found!");
	}
}

...which is better?

DISCLAIMERThis is just my opinion, and different people have different coding preferences. Some people disagree with foreach loops in general.

But I don't disagree with foreach loops. They are really useful when it comes to iterating through arrays.
1. It's much quicker to type
If you want to be as shorthand and thoughtless as possible, then you can just do:
foreach(var item in collection) {
You don't really need to think of a creative name for it, because, after all, in a normal for loop, you would call the variable i.
2. No more array[i]
Instead of constantly having to do array[i], you can just do item, which, when paired with Visual Studio, can be shortened to it and then pressing tab. If you want. But still, it's much quicker than having to write out your array name, then [i].
3. It's a bit easier to understand
Maybe. This may or may not be just to get 3 advantages instead of just 2.
