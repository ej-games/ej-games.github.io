Constructors

This is a C# (with references to Unity) tutorial about constructors. This means that it will be able to be followed in vanilla C#, but will also contain context for Unity users.

What is a constructor?

A constructor is a way of creating (instantiating) a class. If you're using Unity, then constructors can only be used when the class is not using MonoBehaviour.
Look at the top of your class. Is it using MonoBehaviour?

public class ExampleClass : MonoBehaviour {
    // Stuff goes here
}

The class shown above is using MonoBehaviour, which means that you can't use constructors to instantiate it.
Instead, to instantiate this class, add it as a component to an object, and use GameObject.Instantiate().

A constructor is used to instantiate a class that just holds functions and variables, and doesn't run on its own.
For example, if you wanted to store a list of cars (wow, what a creative idea!) then you would do: 

public class Car {
	public string make;
	public string model;
	public float speed;
	// Etc
}
instead of doing an if statement determining the make, model and speed of each car name.

How to create a constructor

Right now, our Car class is useless, because we haven't actually created any instances of it.
This is where constructors come in.

To create a constructor, just do:

public Car() {

}

This will create an empty constructor that takes no arguments. You will just have a car with a blank make, a blank model, and a speed of 0.

So, can you guess what we do next? We add arguments!

public Car(string make, string model, float speed) {

}


So now we have a constructor for a Car with a make argument, a model argument and a speed argument.
But right now, the values in each instance of our Car will be the same. We need to give the car's make, model and speed a value, based on the parameters we give the constructor.

So, in the constructor, we write:

this.make = make;
this.model = model;
this.speed = speed;
Notice the this before each variable: this references the class's variables, instead of the functions. This is only needed because the arguments have the same name as the variables in the class.

The above code will set the make, model, and speed, to the values we pass it, giving each class unique values.

How to instantiate a class

That's all very well and good, but right now, you probably have no clue how to actually use the constructor to make a class.

Here's the simple answer:

Car myCar = new Car("Ford", "Fiesta", 130);


But it's also useful to understand how it works.

When you create a class, you are creating a 'type' of object, that can hold values for each instance.
Putting new Car("Ford", "Fiesta", 130) will create a new car, with a make of 'Ford', a model of 'Fiesta', and a speed of 130.
If, instead, you wanted to make a VW Polo, then you would put new Car("Volkswagen", "Polo", 151)
Because a VW Polo (in 2022) has a top speed of 151mph. Fun fact.

However, if you wanted to compare two classes, you couldn't do:

Car polo = new Car("Volkswagen", "Polo", 151);

if(polo == new Car("Volkswagen", "Polo", 151)) {
Console.WriteLine("Yippee! It worked!");
}


Because, although the values are the same, the items being compared, polo and new Car("Volkswagen", "Polo", 151), are different instances of the 'car' class. The result would never be:
> Yippee! It worked!

Conclusion

So, a constructor is a way to instantiate a class.
To create a constructor, do public ExampleClass(string example).
To create an instance of that class, do new ExampleClass("Example").
