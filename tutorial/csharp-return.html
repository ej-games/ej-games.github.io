Function Returning

This is a C# (with references to Unity) tutorial about returning values in functions. This means that it will be able to be followed in vanilla C#, but will also contain context for Unity users.

Functions as you may know them

If you're using Unity (which a lot of C# users are), then you are probably over-familiar with the word void.

It's everywhere.

void Start(), void Update(), private void OnCollisionEnter(Collision collision).

But you're a big boy (or girl) now, you should really be learning what void actually means. Pff. Kids these days.

Return types

The word void just means "don't return anything". In Start(), for example, you're not trying to get the number of objects in the scene, or 'return' how many times you've entered the scene in the current session. You just want the code to be executed, and to be done with it.

Sometimes, however, that's not what you want. What if you want a function that gives you a random number between 1 and 100? You wouldn't want to do: 
int randomNumber = Random.Range(1, 100); // Unity

var random = new Random();
int randomNumber = random.Next(1, 101); // Vanilla C#
every time you wanted a random number. Especially if you're in vanilla C#.

Instead, you would want to create a function for it. Let's call it GetRandomNumber.

And here's your amazing new class: 
public class Example
{
	public int randomNumber;

	static void Main(string[] args)
	{
		GetRandomNumber();
		Console.WriteLine(randomNumber.ToString());
	}

	public void GetRandomNumber()
	{
		randomNumber = // and then your random number generator
	}
}
But, as are all of the first examples in these tutorials, this isn't the best way to do it.

This is where returning comes in.
Step One
Create your function as if it were a normal int variable. 
public int GetRandomNumber;
Step Two
Add your favourite brackets to the end of the name! 
public int GetRandomNumber()
Step Three
Continue the ritual by adding the final curly brackets of light!! 
public int GetRandomNumber()
{

}
Step Four
The ritual is not yet complete! Add your code into the function! 
public int GetRandomNumber()
{
	var random = new Random();
	int randomNumber = random.Next(1, 101);
}
At this point, if you're using Visual Studio, or another debugger, then it'll get angry at you, and say something like:

Error CS0161: 'Example.GetRandomNumber()': not all code paths return a value

(actually, if you're using the same names, this is exactly the error you'll get)

So, what you need to do is:
Step Five
DON'T GIVE UP! THE RITUAL IS ALMOST COMPLETE! JUST ADD THE FINAL HOLY WORD: 
return randomNumber;
and that's how you make a function return a value.

Explain more!!!

OK!

Remember the function that we made before?
Well, if you were following the entire tutorial, then you should: 
public int GetRandomNumber()
{
	var random = new Random();
	int randomNumber = random.Next(1, 101);
}
You can put this in your Main function to print the random number! Because when you call/invoke the function, it just returns the value you programmed it to, just like it was a normal variable. 
static void Main(string[] args)
{
	Console.WriteLine(GetRandomNumber().ToString());
	/*
		Assuming that the function was static, which I forgot to do, but it's fine,
		as long as it still makes sense
	*/
}
And, because it's just like any other variable, you can make it return ANYTHING YOU WANT! 
public string HelloWorld()
{
	return "Hello world!";
}

public string IsFourGreaterThanFiveOrIsItLess()
{
	if(4 > 5)
	{
		return "It's greater.";
	}
	else
	{
		return "It's less.";
	}
}

public bool IsJohnGoldAmazing()
{
	string response = Console.ReadLine("Is John Gold amazing? [Y/N]");
	if(response == "Y")
	{
		return true;
	}
	else
	{
		return true; // Because John Gold is amazing anyway
	}
}
