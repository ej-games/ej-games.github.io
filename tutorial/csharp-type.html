What are Types?

Stupid question, isn't it? Actually, not as much as you think.
Types are not just your regular ints or strings, or even your custom classes like, for example, Weapon. They are their own special class that has a variant for every other class you make.

That's some bad explaining. Here's an example of just a weapon class with just one field, and a constructor (tutorial here): 
public class Weapon
{
	public int damage;


	public Weapon(int damage)
	{
		this.damage = damage;
	}
}
The class is called Weapon, so you could do

Weapon ak47 = new Weapon(10); to make a weapon that does 10 damage. If you had actually been bothered to make it deal damage.

But, if you wanted to get the actual Type, for whatever reason, you wouldn't be able to just do something like var weaponType = Weapon; because that (for some reason) doesn't work.

Instead, you would do either:
var weaponType = typeof(Weapon);
or
var weaponType = Weapon.GetType(); Which will return a Type class, not a Weapon class.

How do you use Type Arguments?

Glad you asked! You didn't really, did you? Whatever.
You use them just as I did: put the Type (T) in the < and >.

In this section of the tutorial, I will just show you how to use them! If you already know how to use them (ahem, Unity users?) then you can just go on ahead to the section below on how to make methods with type arguments.
But, if you don't have any idea how to use a type argument (or just want some more explanation), then keep reading!

Take the Unity example above: GetComponent<T> - you would usually use it like this, where AudioSource is the type: 
void Start()
{
	AudioSource loadIn = player.GetComponent<AudioSource>;
	loadIn.Play();
}
Simple enough, isn't it? Just put your type in the triangular brackets.
Just saying, there is an alternative, and it isn't pretty: 
AudioSource loadIn = player.GetComponent(typeof(AudioSource)) as AudioSource;
That's just a pain.

How do you make a type argument?

Did you skip the above, or read it because you didn't know? Leave a comment in the comment section down below. (Poetry Corner)

Poetry aside, I will now tell you, step by step, how to make a type argument, and what not to do.
Step 1 - Make your function
This part is simple: just make a function. Or "method", or whatever. Here's an example, and I'll make it as multi-purpose as possible so you can copy and paste to your needs. 
public class Lettuce
{
	public void TransmutateLettuce()
	{
		
	}
}
Now that you've got your lettuce transmutator, we can move on to the next step:
Step 2 - Add triangular brackets
We've done it before above, but this time, we'll actually be adding the T in ourselves! 
public void TransmutateLettuce<T>()
{

}
Now, the T can be used just like any other variable - it effectively is a variable, but with a set datatype of Type.
Step 3 - Add your base type
What does that mean, then? It means we need to put in something to make sure that all of our T arguments View tutorialinherit from a specific class. We don't want to have lettuce transmutating into a string or something.
So, assuming we have some other classes like BabyGem or Iceberg (inheriting from Lettuce): 
public void TransmutateLettuce<T>() where T : Lettuce
{

}
In English, this function says
"Transmutate the lettuce into THIS (T) type, where T must be some sort of lettuce."
Step 4 - Do the actual function
I have no idea how to transmutate a lettuce in C#, so I'll just give you some examples of how to use your new Type argument. 
public void TransmutateLettuce<T>() where T : Lettuce
{
	// Create a new class with the Type argument:
	var newClass = new T();

	// Make a variable with the Type argument:
	T myVariable = ...;

	// You can't perform any functions on the Type argument, because it's a Type, not an instance of the class.
	// (I believe) you can call static methods, assuming the base class has static methods:
	T.InitialiseLettuce();
}
