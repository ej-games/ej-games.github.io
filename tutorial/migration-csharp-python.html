C# -> Python

This is a tutorial that is designed to help you migrate from C# to Python, or just find out the differences.

Intro

C# and Python are wildly different programming (and scripting!) languages. Not just in how they are actually run, but also how they are formatted.

I'm sorry, but if you thought this migration would be as easy as going from Java to C#, then you might be wrong. Python was made for speed, not for ease of use. Which is why "def" is used for functions, rather than the return type.

Overall Thingies

Unlike C#, in Python, you barely use any curly brackets. Any "body" of a statement or function is declared by an indent.
Here's a function in C#, compared to a function in Python: 
public void ThisIsLovely()
{
	Console.WriteLine("Hello!");
}

def this_is_lovely():
	print("Hello!")

Python is also run from top to bottom. This is a small thing but very important to know, because it means that you can't invoke functions that are declared below.

Here are some commonly-used methods in C# and their equivalents in Python:

Set x to user input var x = Console.Read(); x = input()
Wait for x seconds Thread.Sleep(x * 1000); (requires System.Threading;) time.sleep(x) (requires import time)

Variables and Datatypes

Python is weak-typed (variables don't have a set type)! Good in some ways, bad in others, in my opinion.
In Python, you can do this: 
lovely_variable = 4
lovely_variable = "x"

And, from my experience of other peoples' Python code, the naming convention is snake_case, like john_gold and is_your_name_joseph or something like that.

Variables are just declared by putting the variable name and then the value, with no "var" or datatype or anything like that.

Functions and Classes

As shown earlier, this is a function in Python: 
def funky_function():
	print("This is funky and swish!")

But, not as shown earlier, this is a class in Python: 
class CoolTerms:

	swish = "That's very swish!"

	def funky_function(self)
		self.swish = "I prefer 'funky'"

Whatever you do, DO NOT forget the self! Doing that would make your IQ instantly halve.
The first argument in a class function has to be a reference to the class, but does not have to be called "self".

To return a value from a function, you can either just return the value at the end, or you can specify it in the function: 
def funky_function() -> str:
	return "I prefer 'funky'"
