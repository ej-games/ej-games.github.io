Python -> C#

This is a tutorial that is designed to help you migrate from Python to C#, or just find out the differences.

Intro

Python and C# are very different languages. This tutorial was made so you can make links between your existing Python knowledge and how C# works.

Overall Thingies

In Python, as you probably already know, pretty much everything (e.g. if statements, functions, classes) relies on indents, like this: 
def start_game():
	started = True
But in C#, pretty much everything relies on curly brackets, { and }.
And, unlike Python, C# was not made to be quick and compact (you can tell): 
public void StartGame()
{
	bool started = true;
}
And, like many other languages, you have to end every line with a semicolon, apart from a line that declares a body (if statements, functions, classes etc.).

Here are some commonly-used methods in Python and their equivalents in C# (note: "using" is the C# equivalent of "import"):
FunctionPythonC#
Print the message x print(x) Console.WriteLine(x); (requires using System;)
Set x to user input x = input() Console.Read(); (requires using System;)
Wait for x seconds time.sleep(x) (requires import time) Thread.Sleep(x * 1000) (requires using System.Threading;)

Variables and Datatypes

C# is strong-typed (unlike Python) so, when creating a variable, you have to give it a type! 
int x = 4;
x = "Four";
results in
Error CS0029: Cannot implicitly convert type 'string' to 'int'


The naming convention for variables is camelType (var myVariable), and for classes and methods, it is PascalCase (class MyClass, void MyFunction())

Functions and Classes

This is a function in C#: 
void FunkyFunction()
{
	Console.WriteLine("This is funky and swish!");
}
The "void" is the return type - don't return anything.
And this is a class in C#: 
class CoolTerms
{
	string swish = "That's very swish!";

	void FunkyFunction()
	{
		swish = "I prefer 'funky'"
	}
}

To return a value from a function, you just change the "void" to the return type: 
string FunkyFunction()
{
	return "I prefer 'funky'";
}

bool IsSwish(string str)
{
	if(str.ToLower().Contains("swish"))
	{
		return true;
	}
	else
	{
		return false;
	}
}

Other things

In C#, you inherit classes like this: 
class Inherited : BaseClass
{

}

Most things in C# also have access modifiers, which means they can (or can't) be accessed from other classes.
Here are some commonly-used ones:
When applied to classes
Allows for the class to be accessed from any assembly, as long as it has an assembly reference.

When applied to methods and fields
Allows for the field to be accessed outside of the class.
This is the default access modifier when nothing is added.

Cannot be applied to classes

When applied to methods and fields
Prevents the field from being accessed outside of the class.
When applied to classes
Prevents there from being more than 1 instance of the class - all members must be static as well.

When applied to methods and fields
Prevents there from being more than 1 instance of the field, and allows it to be accessed anywhere without a reference to an instance of the class.
When applied to classes
Makes the class only usable if it is inherited by another class - cannot be used like normal.

When applied to methods and fields
The field or method has no value until it is inherited and then can be accessed.
This is the default kind of class (when no other modifier is added), and it is only accessible from the same assembly.
It can also be added to fields, causing roughly the same thing to happen.
